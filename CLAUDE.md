# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Sphinx documentation site for SMACC2 (State Machine Asynchronous C++ for ROS2), hosted at [smacc2.robosoft.ai](https://smacc2.robosoft.ai). Uses `sphinx_rtd_theme` with `dirhtml` builder on Read the Docs.

## Build Commands

```bash
# Install dependencies
pip install -r docs/requirements.txt

# Build HTML locally
cd docs && make html
# Or directly:
sphinx-build -b html docs/source docs/build/html

# Clean build
cd docs && make clean && make html
```

The site is auto-built by Read the Docs on push to `main` (configured in `.readthedocs.yaml`).

## Architecture

- **`.readthedocs.yaml`** — RTD build config (Ubuntu 22.04, Python 3.10)
- **`docs/source/conf.py`** — Sphinx configuration (theme, extensions, sitemap, analytics settings)
- **`docs/source/index.rst`** — Root document with hidden toctree defining page order
- **`docs/source/*.rst`** — Content pages (reStructuredText), each with `.. title::` and `.. meta:: :description:` tags
- **`docs/source/concepts/`** — Concepts subdirectory with `index.rst`, `basics.rst`, `hsm-architecture.rst`, `substate-architecture.rst`
- **`docs/source/tutorials/`** — Tutorial pages (10 tutorials)
- **`docs/source/how-to/`** — How-to guides (Nav2, MoveIt2, PX4, debugging, testing, core behaviors)
- **`docs/source/images/`** — Static images referenced from RST files
- **`docs/source/_static/custom.css`** — Custom CSS overrides (inline code literal color, nav styling, h2 section margins)
- **`docs/source/_static/robots.txt`** — Robots file with sitemap reference (copied to build root via `html_extra_path`)
- **`docs/source/_templates/layout.html`** — Extends RTD theme: custom `| SMACC2` title separator, GA4 + Matomo analytics in footer
- **`docs/requirements.txt`** — Python deps: `sphinx==7.1.2`, `sphinx-rtd-theme==1.3.0rc1`, `sphinx-copybutton`, `sphinx-sitemap`

## How-To Page Heading Structure

All how-to pages for client libraries follow a standard three-section structure:

```
How to Use [X] with SMACC2             (= heading)
├── Getting Started                     (- heading)
│   ├── Required Installations          (~ heading)
│   ├── Assembling the Workspace        (~ heading)
│   ├── Building the Workspace          (~ heading)
│   └── Launching the Application       (~ heading)
├── Tour of the [X] Client Behavior Library  (- heading)
│   ├── Folder Structure                (~ heading)
│   ├── Components                      (~ heading)
│   └── Behaviors                       (~ heading)
└── Using the [X] Client Behavior Library    (- heading)
    ├── Configuring the Orthogonal      (~ heading)
    ├── Using a Behavior                (~ heading)
    ├── Writing Your Own Behavior       (~ heading)
    └── Writing Your Own Component      (~ heading)
```

Additional library-specific subsections (e.g., Planner Presets, SOFT Requirement Pattern) may be added under the appropriate parent section.

## RST Conventions

- Every page has `.. title::` and `.. meta:: :description:` directives for SEO
- Section spacing is handled by CSS (`custom.css` h2 margins) — do NOT use RST `|` pipe lines for vertical spacing
- Use `-` underlines for top-level sections, `~` for subsections
- Code examples should be drawn from real state machines in `smacc2_sm_reference_library` or real client libraries
- Follow the SMACC2 naming convention (`Sm`, `Ms`, `St`, `Or`, `Cl`, `Cb`, `Cp` prefixes) in all examples
- Image paths from subdirectories use absolute paths (e.g., `/images/foo.jpg`)

## SEO & Analytics

- Title tags use `| SMACC2` suffix (via `{% block htmltitle %}` override in `layout.html`)
- `html_title = 'SMACC2'` in conf.py controls the suffix text
- Sitemap generated by `sphinx-sitemap` at `/pages-sitemap.xml` (custom filename avoids collision with RTD's auto-generated `/sitemap.xml`)
- Google Analytics (GA4) ID: `G-NFBR33BNVT`

## Content Strategy

This site follows the [Diátaxis framework](https://diataxis.fr/) — tutorials, how-to guides, concepts, and reference — modeled after [moveit2_tutorials](https://moveit.picknik.ai) and [BehaviorTree.CPP docs](https://www.behaviortree.dev/).

### Core Client Behaviors & Components

The SMACC2 core library includes reusable client behaviors at `smacc2/include/smacc2/client_behaviors/` (cb_call_service, cb_ros_launch, cb_sequence, cb_sleep_for, cb_wait_action_server, cb_wait_topic, cb_wait_topic_message, cb_subscription_callback_base, cb_service_server_callback_base, cb_wait_node). These are not tied to any specific client library and can be used in any state machine. Documentation for these should be included in the tutorials and/or a dedicated reference section.
