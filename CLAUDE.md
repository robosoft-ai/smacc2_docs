# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Sphinx documentation site for SMACC2 (State Machine Asynchronous C++ for ROS2), hosted at [smacc2.robosoft.ai](https://smacc2.robosoft.ai). Uses `sphinx_rtd_theme` with `dirhtml` builder on Read the Docs.

## Build Commands

```bash
# Install dependencies
pip install -r docs/requirements.txt

# Build HTML locally
cd docs && make html
# Or directly:
sphinx-build -b html docs/source docs/build/html

# Clean build
cd docs && make clean && make html
```

The site is auto-built by Read the Docs on push to `main` (configured in `.readthedocs.yaml`).

## Architecture

- **`.readthedocs.yaml`** — RTD build config (Ubuntu 22.04, Python 3.10)
- **`docs/source/conf.py`** — Sphinx configuration (theme, extensions, sitemap, analytics settings)
- **`docs/source/index.rst`** — Root document with hidden toctree defining page order
- **`docs/source/*.rst`** — Content pages (reStructuredText), each with `.. title::` and `.. meta:: :description:` tags
- **`docs/source/images/`** — Static images referenced from RST files
- **`docs/source/_static/custom.css`** — Custom CSS overrides (inline code literal color, nav styling)
- **`docs/source/_static/robots.txt`** — Robots file with sitemap reference (copied to build root via `html_extra_path`)
- **`docs/source/_templates/layout.html`** — Extends RTD theme: custom `| SMACC2` title separator, GA4 + Matomo analytics in footer
- **`docs/requirements.txt`** — Python deps: `sphinx==7.1.2`, `sphinx-rtd-theme==1.3.0rc1`, `sphinx-copybutton`, `sphinx-sitemap`

## Document Tree (defined in index.rst toctree)

`getting-started` → `concepts/index` → `tutorials/index` → `how-to/index` → `repositories` → `debians` → `demos` → `api-documentation` → `troubleshooting` → `about` → `research-group` → `other-resources` → `citations`

The substantive technical content is organized under `concepts/`:
- `concepts/basics.rst` — state functions, transitions, function call order, naming conventions, folder structures, threading model, SmaccSignal
- `concepts/hsm-architecture.rst` — hierarchical states, mode states, super states, inner state loops, deep history, state-local storage
- `concepts/substate-architecture.rst` — object lifetimes, orthogonals, events, clients, client behaviors, components, component-based architecture, state reactors

Most other pages are link collections or embedded video demos.

## SEO & Analytics

- Every page has `.. title::` and `.. meta:: :description:` directives
- Title tags use `| SMACC2` suffix (via `{% block htmltitle %}` override in `layout.html`)
- `html_title = 'SMACC2'` in conf.py controls the suffix text
- Sitemap generated by `sphinx-sitemap` at `/pages-sitemap.xml` (custom filename avoids collision with RTD's auto-generated `/sitemap.xml`)
- Google Analytics (GA4) ID: `G-NFBR33BNVT`

## Content Strategy

This site follows the [Diátaxis framework](https://diataxis.fr/) — tutorials, how-to guides, concepts, and reference — modeled after [moveit2_tutorials](https://moveit.picknik.ai) and [BehaviorTree.CPP docs](https://www.behaviortree.dev/). The planned content structure:

### Tutorials (planned)

**Basic** (using sm_atomic):
1. Your First State Machine — clone, build, run sm_atomic, observe in RTA
2. Adding States & Transitions — add a third state, wire transitions
3. Orthogonals & Concurrent Behaviors — add a second orthogonal
4. Navigation with sm_nav2_gazebo_test_1 — run nav2 state machine in Gazebo, modify waypoints

**Intermediate** (creating custom elements):
5. Client Behaviors — write sync and async behaviors, postSuccessEvent/postFailureEvent
6. Components — create an ISmaccComponent (subscriber + data store pattern)
7. Creating a Client — build an ISmaccClient using the orchestrator pattern
8. Events & State Reactors — typed events, complex transition logic

**Advanced** (complex architectures):
9. Super States with SsSequence — hierarchical state grouping
10. Multi-Stage Missions — full autonomous mission combining multiple orthogonals

### How-To Guides (planned)

- How to use MoveIt2 with SMACC2 (cl_moveit2z)
- How to use Nav2 with SMACC2 (cl_nav2z)
- How to use PX4 with SMACC2 (cl_px4_mr)
- How to use core client behaviors (smacc2/client_behaviors: cb_wait_topic, cb_call_service, cb_ros_launch, cb_sleep_for, etc.)
- How to debug with the SMACC RTA
- How to test a state machine

### Core Client Behaviors & Components

The SMACC2 core library includes reusable client behaviors at `smacc2/include/smacc2/client_behaviors/` (cb_call_service, cb_ros_launch, cb_sequence, cb_sleep_for, cb_wait_action_server, cb_wait_topic, cb_wait_topic_message, cb_subscription_callback_base, cb_service_server_callback_base, cb_wait_node). These are not tied to any specific client library and can be used in any state machine. Documentation for these should be included in the tutorials and/or a dedicated reference section.
